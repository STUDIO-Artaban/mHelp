Interface de Configuration: Onglet "Commun"

L'onglet "Commun" de l'interface de configuration, vous permet de choisir le comportement à suivre dans le cas où le contrôle en cours de configuration, serait invisible ou inactif durant le lancement de la macro. Il vous permet également de définir le message qui sera affiché dans cette éventualité.


Etat L'onglet "Etat" vous permet de configurer la valeur du paramètre du contrôle, s'il s'agit d'un contrôle paramétrable (voir Description des Contrôles et des Types de Paramètres). Dans le cas contraire, cet onglet n'est pas visible.
A Propos Onglet contenant des informations sur le contrôle sélectionné.
Message contrôle non prêt Message affiché dans l'éventualité où le contrôle serait invisible ou inactif.
Message affiché & arrêt du processus Option qui vous permet de choisir le comportement à suivre dans le cas où le contrôle serait invisible ou inactif, et qui consiste à afficher le message ci-dessus, avant d'arrêter le processus DynHelp.
Message affiché & poursuite du processus Option qui vous permet de choisir le comportement à suivre dans le cas où le contrôle serait invisible ou inactif, et qui consiste à afficher le message ci-dessus, avant de poursuivre le processus DynHelp.
Pas de message & poursuite du processus Option qui vous permet de choisir le comportement à suivre dans le cas où le contrôle serait invisible ou inactif, et qui consiste à ne pas afficher le message ci-dessus, et de poursuivre le processus DynHelp.
Attendre contrôle actif & affiche message (Max délai) Option qui vous permet de choisir le comportement à suivre dans le cas où le contrôle serait invisible ou inactif, et qui consiste à afficher le message ci-dessus, avant de poursuivre le processus DynHelp.
Aide (F1) Permet de lancer l'aide utilisateur du système DynHelp.
Appliquer Permet d'appliquer les changements configurés pour le contrôle sélectionné.
Fermer Permet de fermer l'interface.

Voici ci-dessous l'onglet "Commun" de l'interface de configuration, lorsque le contrôle sélectionné est de type ListBox, ListView, TreeView, etc... (voir Description des Contrôles et des Types de Paramètres). Le point commun entre ces contrôles est que leur état dépend d'une sélection.


Message échec de la sélection Message affiché dans l'éventualité où le processus DynHelp est arrêté (voir option ci-dessous), car il n'est pas parvenu à sélectionner l'élément souhaité par l'utilisateur (cet élément n'est pas contenu dans le contrôle).
Message & arrêt si le changement de sélection à échoué Option permettant d'arrêter le processus DynHelp dans le cas où il ne parviendrait pas à changer l'état du contrôle, car l'élément à sélectionner n'est pas contenu dans ce contrôle.

Remarques

L'option qui consiste à attendre que le contrôle soit visible ou actif avant de poursuivre le processus DynHelp, dépend de la configuration du délai d'attente maximum entre deux actions (voir Interface Principale: Onglet "Configuration"). Une fois ce délai dépassé, le message est alors affiché avant d'arrêter le processus DynHelp.

Les options permettant de définir le comportement à adopter lorsqu'un contrôle n'est pas visible ou désactivé, ne sont pas autorisées pour un "contrôle" de type "change d'application", car toutes actions sur ce logiciel ne seront pas réalisables si le système DynHelp ne parvient pas à le lancer, ce qui limite le comportement à suivre, à l'affichage du message et à l'arrêt du processus DynHelp.
D'autres contrôles comme les Boutons ou les Menus, ont également les options de poursuite de processus désactivées, car les actions réalisées par ce type de contrôle sont en générale liées à des changements au niveau du comportement et de la gestion de l'interface, rendant impossible la poursuite du processus de lancement de macros.
