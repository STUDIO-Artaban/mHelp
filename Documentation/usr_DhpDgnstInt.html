DynHelp: Guide Utilisateur V1.21
Interface de l'Outil "DhpDgnst.exe"

L'outil "DhpDgnst.exe" permet de restaurer à l'état initial certaines clés de la base de registres, générées par le système DynHelp durant un processus de création ou de lancement de macro.


Application Nom de l'application utilisant le système DynHelp.
Statut Statut de l'application : Ouvert / Fermé / Non trouvé. Ce dernier statut signifie que l'application a été détruite ou déplacée de son répertoire d'origine.
Système DynHelp Statut du système DynHelp dans cette application : Fermé / Interface ouverte / Macro en cours de création / Macro en cours de lancement.
Liste des applications Liste contenant des informations sur le statut des applications utilisant le système DynHelp.
Rafraîchir Permet de rafraîchir la liste des statuts des applications, et le statut du système DynHelp.
Détruire Permet de détruire les clés de la base de registres liées à l'application sélectionnée, dont le statut est 'Non trouvé' (voir Remarques).
Statut de DynHelp Statut général du système DynHelp : En attente / En cours de création de macro / En cours de lancement de macro / En cours de sauvegarde de macro.
Aide (F1) Permet de lancer l'aide utilisateur du système DynHelp.
Initialiser Permet de restaurer l'état initial des clés de la base de registres (voir Remarques).
Fermer Permet de quitter cet outil.

Remarques

Détruire les clés de la base de registres liées à une application qui a été détruite ou déplacée de son répertoire d'origine, a pour effet de supprimer la configuration, les fichiers de macros ouvert et les macros favorites de cette application.

Restaurer l'état initial des clés de la base de registres revient à considérer que toutes les applications utilisant le système DynHelp sont fermées (comme il vous est demandé de le faire à l'ouverture de cet outil), et que par conséquent aucune interface principale DynHelp n'est ouverte, et que le système est en attente.

Une fois l'initialisation terminée, un message d'avertissement vous invite à fermer cette application avant de reprendre vos opérations de création ou de lancement de macros. Si toutefois vous laissez cet outil ouvert, ou que vous ne fermez pas toutes les applications utilisant le système DynHelp à l'ouverture de ce dernier, cela n'a pas d'importance, mais dans ce cas il est déconseillé de restaurer l'état initial des clés de la base de registres, sans quoi d'autres problèmes peuvent survenir comme notamment au niveau de la gestion des macros favorites.
