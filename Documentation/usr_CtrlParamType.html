Remarques

Les contrôles de type RadioBox et ToolBar radiobox ont un état dont le paramètre est un Entier (voir le chapitre correspondant aux contrôles de ce type dans l'Interface de Configuration: Onglet "Etat" de Type Entier).

Le "contrôle" nommé "Change d'application" n'est pas un contrôle mais une action, correspondante à un changement d'application. En effet, DynHelp vous permet de créer une macro passant d'un logiciel à un autre, dans la mesure où ces logiciels supportent le système DynHelp, et que l'application qui suit autorise la partage de ses fonctionnalités.
Pour ce faire, vous devez lancer cette dernière application une fois le mode création de macro activé, car si celle-ci est déjà ouverte lors de l'activation de ce mode, changer d'application en l'activant n'aura aucun effet. Une fois l'application lancée, le mode création de macro étant actif, vous pourrez alors passer de cette application à l'application d'origine, et inversement, en changeant simplement leur activation.
